DE IK Nyári Ösztöndíj Felhívás

Hallgatót keresünk az underdocs nyílt forráskódú dokumentáció-generátor továbbfejlesztéséhez.

Az underdocsról

A következőkben szeretnénk röviden bemutatni, hogy mi is az underdocs, miért jött létre, milyen célokkal kezdtük el fejleszteni.

Röviden a dokumentáció-generátorokról

A szoftverfejlesztésnek elengedhetetlen része a tényleges programozáson felül a dokumentáció készítése is. Megfelelő dokumentációval nemcsak karbantarthatóbbá tesszük a kódbázist, hanem a projektünket felhasználó kliensek is könnyebben eligazodnak rajta.

Elősegítendő a kódbázis egyszerű felfedezhetőségét és felhasználhatóságát, számos olyan eszköz készült, mely a megfelelő kommenteket felhasználva (úgynevezett doc commentek) HTML vagy más formátumú kimenetet készít. Java fejlesztők számára ismert lehet a javadoc eszköz, míg JavaScriptben jártasak a JSDoc formátumról hallhatták már.

A dokumentáció és a C

A C minden idők egyik legnépszerűbb programozási nyelve. Lényegében nem létezik olyan platform, melyhez ne létezne C fordító. Beágyazott és teljesítménykritikus alkalmazások esetén ma is a C az elsődlegesen használt programozási nyelv.

Népszerűsége ellenére, a C nem rendelkezik hivatalos ajánlással, ami a dokumentációs kommentek formátumát illeti. Ehelyett a fejlesztőknek kell kiválasztaniuk az általuk használni kívánt formátumot és generátort. A teljes igénye nélkül, népszerűbbnek tekinthetők a következők:

  • doxygen,
  • cldoc,
  • standardese,
  • QDoc,
  • Sphinx (Breathe segítségével).

Miért kezdtünk bele egy n + 1-edik eszközbe?

Az elmúlt években létrehoztunk egy közepes méretű kódbázist (több mint 50 header fájl) a CryptID személyében, mely egy C-ben írt, WebAssemblyt célzó kriptográfiai könyvtár. Az elmúlt hónapokban megnőtt az érdeklődés a könyvtár iránt, amire mi szerettünk volna egy felhasználóbarát dokumentációval válaszolni.

Ennélfogva, megnéztük, milyen lehetőségeink vannak, ha szép HTML dokumentációt szeretnénk készíteni a kódbázisunkból. Az előbb felsorolt eszközökön végighaladva a következő hiányosságokat tapasztaltuk:

  • Az eszköz igazából C++-hoz készült, a C támogatása mellékes, csak amolyan bónusz.
  • A generált oldalak gyakran úgy néznek ki, mintha a '90-es évek végén hozták volna létre őket.
  • A létrehozott HTML weboldal nehezen böngészhető, nehéz rajta eligazodni.
  • Kényelmetlen a dokumentációs formátum, nehéz megfelelően formázott, akár képeket és matematikai formulákat is tartalmazó dokumentációt hozzáadni.

Mivel nem találtunk olyan eszköz, ami kielégíti az igényeinket, úgy döntöttünk, készítünk egy sajátot. Ez lett az underdocs. Alant látható egy apró példakód és a belőle generált kimenet.

Az elvégzendő feladat

Az underdocs jelenleg inkább egy proof-of-concept alkalmazásnak tekinthető, semmint kész szoftverterméknek. A nyári szakmai ösztöndíj keretében a feladatod annak elősegítése, hogy az underdocs egy funkcionalitásban gazdag, akár production környezetben is felhasználható generátor legyen.

További egyeztetés után a GitHub Issuesban vezetett feladatok közül választjuk majd ki azokat, melyek az ösztöndíj tényleges scope-ját jelentik. Néhány issue ízelítőül:

  • Kódrészletek és matematikai kifejezések előzetes kirajzolása GraalVM segítségével.
  • Dokumentáció generálása ugyanazon projekt több verziójához.
  • Kliens-oldali keresés előzetesen kiszámolt indexek felhasználásával.

Mit kapsz a projekttől?

A projektben való részvétellel a következőkkel lehetsz gazdagabb:

  • Sikeres pályázat esetén a DE IK Nyári Szakmai Ösztöndíj összege.
  • A C nyelv és nyelvtan mélyebb ismerete.
  • A Kotlin nyelv mélyebb ismerete (az underdocs Kotlinban készült).
  • Megismerkedhetsz a JVM-alapú fejlesztés jövőjét jelentő GraalVM-mel.
  • A git és a GitHub (Projects, Issues, PRs, Actions) támogatta fejlesztés elsajátítása.
  • Több hónapon át tartó mentorálásban lesz részed az ipart és konferenciákat is megjárt informatikusok támogatásával.
  • Lesz egy olyan nyílt forráskódú projekt a portfóliódban, melyet bármely állásinterjún büszkén felvillanthatsz.

Mit várunk el Tőled mi?

Mielőtt jelentkezel, kérjük, gondold át, hogy rendelkezel-e a következőkkel:

  • A Java (esetleg Kotlin) stabil ismerete.
  • A HTML és a CSS alapszintű, de magabiztos ismerete.
  • A JavaScript alapszintű ismerete.
  • A nyár folyamán rendelkezésre álló idő (szeretnénk, ha hétről hétre foglalkoznál a projekttel).
  • Vágy a fejlődésre, tanulásra.
  • Képesség az önálló problémamegoldásra. Bár mentorálunk, azonban szeretnénk, ha sok kihívással önállóan is képes lennél megbirkózni.
